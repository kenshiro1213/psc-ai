# 04. スライド構成

## 設計方針（重要）

- **ヒアリングとデモは「扉スライド」のみ。投影せず手元台本＋実機で進行**
- スライドは「教育・セキュリティパート」と「セクション扉」「クロージング」に集中
- **全20枚程度** に削減（投影時の冗長さを避け、デモ・対話に時間を割く）
- デモは Claude for Work のチャット画面を直接投影

---

## 技術スタック

| 項目 | 選定 | 理由 |
|---|---|---|
| フレームワーク | **reveal.js** | HTMLスライドの定番。Chrome印刷でPDF出力可（`?print-pdf`） |
| 記述形式 | Markdown（`<section data-markdown>`） | 編集が楽。差分管理が容易 |
| PDF出力 | Chromeの「印刷」→「PDFに保存」 | `decktape` でも可。Chrome経由が安定 |
| ホスティング | ローカル `python3 -m http.server` | 投影は手元PCで十分 |

## デザイン方針

### カラーパレット（PSC公式サイト準拠）

```
/* PSC Brand-aligned palette */
--psc-navy:        #0A3D62;  /* メインアクセント（見出し・強調・ボタン背景）*/
--psc-navy-dark:   #062840;  /* 深い紺 */
--psc-blue:        #1E6FBA;  /* リンク・サブアクセント */
--psc-bg:          #FFFFFF;  /* 背景 */
--psc-bg-soft:     #F5F7FA;  /* セクション区切り・カードBG */
--psc-text:        #1A1A1A;  /* 本文 */
--psc-text-muted:  #555555;  /* 補足テキスト */
--psc-divider:     #E0E5EC;  /* 罫線 */
--psc-accent-warm: #C8932A;  /* 注意喚起・警告（控えめなゴールド）*/
```

> PSCサイトは「白基調＋紺アクセント」のクリーンなビジネス調。**白×紺×グレー** の3色で統一感を出す。

### フォント

```
/* Body & Headings */
font-family: "Noto Sans JP", "游ゴシック", "Yu Gothic", "ヒラギノ角ゴ ProN", sans-serif;

/* Code */
font-family: "JetBrains Mono", "Menlo", "Consolas", monospace;
```

明朝は使わない（PSCサイトに合わせる）。

### レイアウト原則

- **1スライド1メッセージ**
- 見出しは紺、本文は黒、補足は薄グレー、強調はボールド＋紺
- 文字サイズ：本文28〜32pt、見出し44〜52pt
- 余白を広く、詰め込まない
- ページ番号・ロゴはフッター小さく

## PDF出力手順

1. `cd slides && python3 -m http.server 8000`
2. ブラウザで `http://localhost:8000/?print-pdf`
3. Chromeの印刷 → 詳細設定 → 余白「なし」・背景画像「あり」・用紙「A4横」
4. PDFに保存

---

## デモ実機切替の運用

- スライドアプリ（フルスクリーン）と Claude for Work（ブラウザ）を **2画面 or Cmd-Tabで素早く切替**
- セクション5扉スライドで一呼吸 → Claude for Work タブへ
- 各デモ後、扉スライドに戻らず次デモへ直接（時間節約）
- 全デモ終了後、扉スライドに戻ってからクロージングへ

## 次のアクション（スライド実装）

1. reveal.js雛形セットアップ（`index.html` + テーマCSS）
