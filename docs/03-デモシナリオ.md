# 03. デモシナリオ（18分・3本）

## 共通方針

- **ツールは Claude for Work（Claude.ai のチャットUI）一本**
- **入力はファイル添付**（Excel 中心、Demo B は PDF も）
- フォルダ巡回・スクリプト実行はしない。チャット欄で完結
- すべて **ダミーデータ** で実施。冒頭で明言
- 失敗時の保険：各デモの **成功時録画** を用意
- 進行：扉スライド1枚のみ → 即実機 → 出力確認 → 狙うメッセージ

> サンプルデータは `demo/` 配下に Excel ファイルとして格納。
> プロンプトは [docs/prompts/](./prompts/) にコピペ用テキストを置く。

---

## Demo A：Q1 顧客・交渉履歴の引継ぎ（6分）

### 対応する事前質問
- Q1：異動時の顧客管理や交渉履歴をAIツールでスムーズに引継ぎできないか
- Q10：AIを活用した顧客管理・案件管理の手法

### シナリオ

> 「営業担当の田中さんが異動した。後任の佐藤さんが、顧客A（架空の建設会社）との過去2年間の交渉内容を **5分で把握** して、来週の打ち合わせに臨みたい」

### 入力データ（1ファイル）

`demo/demo-a-handover/顧客A_引継ぎ資料.xlsx`

| シート | 内容 |
|---|---|
| 案件概要 | 顧客名、規模、案件名、フェーズ、担当者、決裁者、予算規模、納期 |
| メール履歴 | 日付・送信者・件名・本文（10〜12通） |
| 議事録 | 日付・参加者・議題・決定事項・宿題（3〜4本） |
| 要望リスト | 要望項目・初出日・最新ステータス・優先度・備考 |
| キーパーソン | 氏名・役職・関係性・コミュニケーション特性 |

意図的に要望の変遷・論点の対立・未解決事項を散らしておく。

### デモの流れ（6分）

| 時間 | アクション |
|---|---|
| 0:00 | シナリオを口頭で説明（30秒） |
| 0:30 | Excel ファイルを開いて中身をざっと見せる（30秒） |
| 1:00 | Claude for Work に Excel を添付 → プロンプト1を投入 |
| 1:30 | 出力を確認しながら、要望の変遷・未解決事項が拾えていることを示す（2分） |
| 3:30 | プロンプト2「次回MTGの質問リストを作って」を追加 |
| 4:30 | 出力確認、優先度付き質問リストを見せる |
| 5:30 | 狙うメッセージ：「整理されたデータなら引継ぎは数分」 |

### プロンプト（[prompts/demo-a-prompt.md](./prompts/demo-a-prompt.md) 参照）

**1回目**：引継ぎサマリ生成
**2回目**：次回MTGの質問リスト生成

### 見せ場

- 1ファイル添付で複数シートを横断的に読む
- 時系列・論点・未解決を構造化して返す
- 追加プロンプト1行で別アウトプット（質問リスト）が出る

### 狙うメッセージ

> **データさえ整理されていれば、引継ぎは数分の作業になる。**
> 「整理されている」が前提 → セクション1-(b) の伏線回収。

## Demo B：Q2/Q3 仕様書 → 社内ドキュメント生成（6分）

### 対応する事前質問
- Q2：見積要綱書、仕様書を読み込み、社内図説シートを自動作成
- Q3：図面（BIM）や仕様書を読み込み、積算数量書を自動作成
- Q4：過去案件データを学習し概算見積（入口として）
- Q6：技術提案書のたたき台（入口として）

### シナリオ

> 「見積要綱書（PDF）と仕様書が届いた。担当者が中身を読んで社内図説シートと積算数量のたたき台を作るのに半日かかっている。これをAIで30分にしたい」

### 入力データ（2ファイル）

```
demo/demo-b-docgen/
  見積要綱書_サンプル.pdf            # 公開資料を加工 or 自作（10〜20ページ）
  社内図説シート_テンプレート.xlsx   # 既存フォーマット模擬
```

`社内図説シート_テンプレート.xlsx` のシート構成：

| シート | 内容 |
|---|---|
| 図説シート | 工種・項目・仕様・数量・単位・備考・出典の列ヘッダのみ（中身は空） |
| 記入要領 | 各列の記入ルール |
| 凡例 | 工種コード・略号の凡例 |

### デモの流れ（6分）

| 時間 | アクション |
|---|---|
| 0:00 | シナリオ口頭説明（30秒） |
| 0:30 | PDFを開いて分量を見せる（30秒） |
| 1:00 | Excel テンプレートを開いて構造を見せる（30秒） |
| 1:30 | Claude for Work に PDF＋Excelテンプレを添付 → プロンプト1を投入 |
| 2:30 | 構造化出力を確認（テーブル形式で表示される） |
| 3:30 | プロンプト2「コンクリート種別の根拠は？」で **トレーサビリティ確認** |
| 4:30 | プロンプト3「Excel形式で出力して」 → 添付ダウンロード可能なファイル取得 |
| 5:30 | 狙うメッセージ：「たたき台を高速生成、人間は確認に集中」 |

### プロンプト（[prompts/demo-b-prompt.md](./prompts/demo-b-prompt.md) 参照）

**1回目**：PDF から構造化抽出
**2回目**：根拠箇所の引用
**3回目**：Excel形式で出力

### 見せ場

- **根拠引用**でハルシネーション検証ができることを示す
- 「不明な箇所は推測しない・要確認と書く」プロンプト制御の効果
- Excelファイル添付ダウンロードまで完結

### 狙うメッセージ

> **AIは作業を置き換えるのではなく、たたき台を高速で作る道具。**
> 人間は確認・修正に集中できる。**0→1の手間が消える**。

### 注意点

- ダミーPDFはOCR済みのテキスト含む形式（画像PDFは避ける）
- 公開された自治体発注仕様書を加工するか、自作する
- 事前リハで読み込み品質・根拠引用精度を確認

## Demo C：Q12 自治体発注公告の絞り込み・整形（6分）

### 対応する事前質問
- Q12：各自治体の発注公告・開札情報をAIに収集させる、精査させる、リスト化、共有

### シナリオ（変更点）

> 「複数自治体の発注公告ページから手作業で集めた30件の公告データがある。これを **PSCの関心条件で絞り込み**、リスト化＋共有用文面まで作りたい。さらに『**毎週これを自動でやるには？**』も相談したい」

> **方針変更**：自動巡回スクリプトの実行はチャットUIではできない。
> → 「収集済みデータを構造化・絞り込み」をデモし、自動化の話は口頭＋Claudeに設計させる。

### 入力データ（1ファイル）

`demo/demo-c-bid-watch/発注公告_収集データ.xlsx`

| シート | 内容 |
|---|---|
| 公告データ | 自治体・公告日・案件名・工種・予定価格・場所・URL・公告本文（30件） |
| 抽出条件 | PSCの関心条件（工種・地域・予定価格レンジ・キーワード） |
| 出力テンプレ | 共有メール用の定型文面（差し込み変数付き） |

### デモの流れ（6分）

| 時間 | アクション |
|---|---|
| 0:00 | シナリオ口頭説明（30秒） |
| 0:30 | Excel の3シートを開いて見せる（30秒） |
| 1:00 | Claude for Work に Excel を添付 → プロンプト1を投入 |
| 2:00 | 絞り込み結果（5〜8件）が表で表示される |
| 2:30 | プロンプト2「Slack共有用の文面を作って」 |
| 3:30 | 共有文面が生成される |
| 4:00 | プロンプト3「毎週月曜に自動でやるには？」 |
| 4:30 | Claudeが Python+cron / GAS / Make / n8n などの選択肢を提示 |
| 5:00 | 「次回PoCで実装します」と着地 |
| 5:30 | 狙うメッセージ：「定期業務の自動化はAIに設計から任せられる」 |

### プロンプト（[prompts/demo-c-prompt.md](./prompts/demo-c-prompt.md) 参照）

**1回目**：条件で絞り込み
**2回目**：共有用文面生成
**3回目**：自動化設計の相談

### 見せ場

- **30件 → 関心案件だけ** に絞り込まれる即時性
- 共有文面まで一気に生成され、運用イメージが湧く
- 自動化の **設計** までAIがやれる（実装は次のステップ）

### 狙うメッセージ

> **AIは「都度使う道具」だけでなく「業務設計のパートナー」にもなる。**
> 定期業務の自動化は次のフェーズの大きなテーマ。
