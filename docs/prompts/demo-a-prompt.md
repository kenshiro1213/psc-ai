## プロンプト1：引継ぎサマリ生成

```
添付の Excel は、PSC（建設会社）の営業担当が異動になったため、
後任向けに過去の交渉履歴をまとめた資料です。

5つのシートがあります：
- 案件概要
- メール履歴
- 議事録
- 要望リスト
- キーパーソン

これらをすべて読み込んで、後任の営業担当（佐藤）が
来週の打ち合わせ（2026年6月2日）に向けて
**5分で全体像を把握できる引継ぎサマリ** を作ってください。

以下の構造で Markdown 出力してください：

1. 顧客概要（社名・規模・関係性）
2. 案件サマリ（何の案件か・現在のフェーズ・予算）
3. 要望の変遷（時系列で重要な節目）
4. 主要な論点・懸念事項
5. **未解決事項・宿題**（最重要）
6. キーパーソン（決裁者・特性・注意点）
7. 競合状況
8. コミュニケーション上の注意点（前任からのアドバイス相当）

注意：
- Excelに書かれていない情報は**推測しない**こと
- 不確かな部分は「要確認」と明記
```

### 期待する出力ポイント

- 要望の変遷で、屋上太陽光（4月）→ 冷蔵区画追加（7月）→ 予算上限交渉（9月）→ 経営層対応（1月以降）の流れが拾えている
- 未解決として **北側住宅街の騒音・振動対策（要望#6）** が出てくる
- 競合C社（49億円）との価格劣勢／技術優位の構図
- 西郷取締役は対面重視・ROI型、加納部長はメール多用・数字重視

## プロンプト2：次回MTG向け質問リスト

```
ありがとうございます。
このサマリを踏まえて、6/2の打ち合わせで佐藤が **確認すべき質問リスト** を作ってください。

以下の条件でお願いします：
- 優先度（高/中/低）を付ける
- 「顧客に直接聞く質問」と「社内で先に確認する質問」に分ける
- 質問の意図（なぜ聞くか）も簡潔に併記
- ヒアリングが膠着しないよう、最大10問程度に絞る
```

### 期待する出力ポイント

- 顧客に聞く質問：北側住宅街への対応進捗、競合C社対比での懸念、最終予算の確定タイミング、屋上太陽光の発電事業者選定状況、など
- 社内に聞く質問：田中前任への確認事項、設計の山本さんへの技術論点、近隣説明会のサポート体制
- 「価格に直接踏み込まない」など、いきなり聞きづらい話題への配慮
