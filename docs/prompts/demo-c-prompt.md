# Demo C プロンプト（発注公告の絞り込み・整形）

> Claude for Work（Claude.ai）チャットに `demo/demo-c-bid-watch/発注公告_収集データ.xlsx` を添付してから順に投入。

---

## プロンプト1：条件で絞り込み

```
添付の Excel は、複数自治体の発注公告ページから収集した30件の案件データです。

シート構成：
- 公告データ：30件の案件（自治体・公告日・案件名・工種・予定価格・場所・URL・本文抜粋）
- 抽出条件：弊社（PSC＝建設会社、PCa工法が強み）の関心条件
- 出力テンプレ：共有メール／Slack文面のテンプレート

「抽出条件」シートの条件で、「公告データ」を絞り込んでください。

出力：
- 関心案件のみのテーブル（マッチした列＋PSC適合理由）
- 適合理由はキーワード・地域・価格レンジなど **なぜマッチしたか** を簡潔に
- 上位 5〜8件で表示
- 除外した理由が興味深い案件があれば末尾に注記
```

### 期待する出力ポイント

- PCa／プレキャスト／物流倉庫／BCP キーワードを含む案件が上位に
- 関東圏案件の優先度が高い表示
- 改修系・小規模・短工期は除外されている
- 「つくば物流倉庫」「江東区民会館」「東洋ロジ系倉庫（実は仙台・舞洲も）」「市原市体育館」あたりが選ばれる想定

## プロンプト2：共有用文面の生成

```
ありがとうございます。
絞り込んだ結果を、「出力テンプレ」シートの文面テンプレートに沿って **Slack共有用の文面** に整形してください。

- {{}} の差し込み変数を実データで埋める
- ハイライト1行コメントには、今週の特徴（PCa系が多い／関東圏に良案件 など）を入れる
- 推奨アクションには、優先的に動くべき案件を1〜2件名指しで提案
- 絵文字は不要、ビジネス調で
```

### 期待する出力ポイント

- 完成した文面がそのままSlackにコピペできる形
- 「今週はPCa適合案件が複数」「物流倉庫案件が3件と豊富」など気の利いた1行
- 推奨アクションで「○○の入札説明会日程を確認」など具体性

## プロンプト3：自動化の設計相談

```
最後に相談です。
今は手作業で各自治体サイトを巡回して、このExcelを作っています。
**毎週月曜の朝に、この一連の作業を自動で動かす** にはどうすればよいでしょうか？

以下を含めて設計案を提示してください：
- 巡回部分（HTML取得・更新検知）の実現方法
- データ抽出・絞り込み部分の実装方法
- 共有部分（Slack／メール通知）の方法
- 推奨技術スタック（Python＋cron / GAS / Make / n8n など複数案）
- それぞれのメリット・デメリット
- PoCの最小構成（1自治体・1工種だけから始める案）
```

### 期待する出力ポイント

- 複数の実装パターンが提示される（Python+cron／GAS／Make／n8n／ローコード等）
- 自治体サイトのスクレイピング規約に触れている
- PoCの最小構成案が現実的（1自治体・1工種から始める）
- 通知連携の選択肢（Slack Webhook、メール、Teams 等）
- 「次回提案の入口」として最適な設計案

→ 「自動化の設計までAIに相談できる」ことを印象付ける
