# Demo B プロンプト（仕様書 → 社内ドキュメント生成）

> Claude for Work（Claude.ai）チャットに以下2ファイルを添付してから順に投入：
> - `demo/demo-b-docgen/見積要綱書_サンプル.pdf`
> - `demo/demo-b-docgen/社内図説シート_テンプレート.xlsx`

---

## プロンプト1：仕様書から構造化抽出

```
添付の2ファイルです：
1. 見積要綱書_サンプル.pdf … 顧客から届いた見積要綱書（物流倉庫の新築）
2. 社内図説シート_テンプレート.xlsx … 弊社（PSC）の社内図説シートのテンプレート

見積要綱書を読み込んで、社内図説シートの「図説シート」シートのフォーマットに合わせて
**たたき台** を作ってください。

ルール：
- 「凡例」シートの工種コード体系と単位略号を使う
- 「記入要領」シートの記入ルールに従う
- 数量・単位を明記
- **仕様書に書かれていない値は推測しない**。「要確認」と明記し、備考に理由を書く
- 各項目の **出典**（仕様書のセクション番号や見出し）を必ず記載

まずは Markdown のテーブル形式で出力してください。
（Excel ファイル化は次のステップで依頼します）
```

### 期待する出力ポイント

- C-01〜C-04（コンクリート系）、S-01〜S-02（鉄骨系）、F-01〜F-03（仕上系）、M-01〜M-03（設備系）、X-01（外構）の各カテゴリで項目が拾えている
- 基礎杭：168本／3,650㎥／FC=30 など定量値が正しく抽出されている
- 冷蔵区画の特殊仕様（A種断熱100mm／+5℃）が拾えている
- 屋上太陽光（180kW）が拾えている
- 推測しなかった項目に「要確認」と明記されている

## プロンプト2：根拠の引用確認（トレーサビリティ）

```
ありがとうございます。
出力した中から、以下3項目について **見積要綱書のどのセクション・どの記述から抽出したか** 教えてください：

1. PCa梁のコンクリート強度
2. 冷蔵区画の床荷重
3. 北側遮音壁の高さ

それぞれ、見積要綱書の該当文を **引用** してください。
もし仕様書に明記されていなければ「明記なし」と答えてください。
```

### 期待する出力ポイント

- 1. PCa梁：「2.1 柱・梁」のPCa梁 FC=45 N/mm² を引用
- 2. 冷蔵区画床荷重：「2.2 床・スラブ」の1F床荷重 1.5 t/㎡（冷蔵区画）を引用
- 3. 北側遮音壁：「5. 外構」の高さ4m を引用

→ 「根拠が出せる＝ハルシネーションを検証できる」ことを来場者に印象付ける

## プロンプト3：Excel ファイル形式で出力

```
たたき台を、添付した `社内図説シート_テンプレート.xlsx` の「図説シート」シートに
直接記入した形で、Excel ファイルとして出力してください。

- 既存の凡例・記入要領のシートは保持
- 不明な値は空欄＋備考に「要確認」
- 出典列にはセクション番号を入れる

ダウンロードできる形でファイルを出力してください。
```

### 期待する出力ポイント

- Excel ファイルがチャットからダウンロードできる形で生成される
- 「図説シート」シートに数十行のデータが入っている
- 「凡例」「記入要領」シートは保持されている
