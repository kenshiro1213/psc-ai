# 02. セキュリティ解説

> ヒアリング中、教育パート（セクション1）の中で **5〜7分** で扱う。資料は本書から抜粋してスライドに反映。

## メッセージの骨子

**「便利だから使いたい」と「機密が漏れたら困る」は両立できる。**
鍵は **①プランの選び方／②データ分類／③社内ルール** の3点。

## 1. データが学習に使われるか？プラン別の整理

### AI事業者のデータ取り扱い（2026年5月時点・要確認）

| プラン | 入力データの学習利用 | 推奨用途 |
|---|---|---|
| ChatGPT Free / Plus | デフォルトで学習対象（オプトアウト可） | 個人利用・機密なし |
| ChatGPT Team / Enterprise | **学習対象外** | 法人利用の標準 |
| Claude Free / Pro | **学習対象外**（ユーザー報告データ等の例外あり） | 個人利用 |
| Claude for Work（Team / Enterprise） | **学習対象外** | 法人利用の標準 |
| OpenAI API（直接利用） | **学習対象外**（デフォルト） | 開発・自動化 |
| Anthropic API | **学習対象外**（デフォルト） | 開発・自動化 |
| Gemini（個人） | 学習対象になり得る | 個人利用 |
| Microsoft 365 Copilot | **学習対象外**（Microsoft 365テナント内で完結） | Office連携が必要な場合 |

> 規約は頻繁に更新される。導入時は各サービスの最新規約を確認すること。

### オプトアウト設定

- 各サービスの設定画面で「データの学習利用」を無効化できる
- ChatGPT：設定 > データコントロール > 「全員のためにモデルを改善する」をOFF
- Claude：プライバシー設定で同様に変更可能
- **問題点**：個人アカウントだと設定が個人任せ → 組織全体での徹底は困難

## 2. インシデント事例

### サムスン電子（2023年）
- **何が起きたか**：社員がChatGPT（個人プラン）に**社内ソースコードと会議議事録**を貼り付けて要約・修正を依頼
- **何が問題か**：機密情報が外部AI事業者のサーバに送信され、学習データに含まれる可能性が発生
- **結果**：社内でのChatGPT利用が**禁止**に

### 国内事例（一般化）
- 顧客の問い合わせ内容を社員が個人ChatGPTで処理 → 個人情報の社外流出
- 個人プランで翻訳・要約 → 取引先情報・契約金額が学習対象に入るリスク
- 「無料だから」「便利だから」と個人が勝手に使う → ガバナンス崩壊

### 建設業界で特に守るべき情報
- **入札情報**（予定価格、入札戦略）
- **見積根拠**（労務単価、外注先、利益率）
- **顧客要望・契約条件**
- **図面・BIM データ**（設計知財）
- **近隣対応履歴**（紛争リスク情報）
- **過去案件の技術ノウハウ**

これらが外部AI事業者の学習データに入ると、競合の質問への回答に断片が漏れ出す可能性がゼロではない。

## 3. 推奨される運用パターン

### レベル1：個人レベルの最低限
- 機密情報を**そのまま貼り付けない**
- 固有名詞・金額・人名を**伏字**にしてから貼る
- 個人プランでオプトアウト設定

### レベル2：チーム・部署レベル
- **法人プラン**（ChatGPT Team / Enterprise、Claude for Work）に統一
- 学習無効化が標準＋管理者によるユーザー管理
- 利用ガイドラインを部署で共有

### レベル3：全社ガバナンス（推奨ゴール）
1. **法人プラン契約**：会社主導で導入。個人アカウント禁止
2. **データ分類**：機密度に応じて「使ってよいAI／だめなAI／加工して使うAI」を線引き
3. **利用ガイドライン整備**：何を入力してよいか／だめか、社員教育
4. **ログ監査**：誰が何を入力したか追跡（法人プランの管理機能 or 専用ゲートウェイ）
5. **PoC・本番でツールを分離**：実験は手元のサンドボックスで、本番は監査ログ付きで

### データ分類例（建設業向け）

| 区分 | 例 | 利用可能なAI |
|---|---|---|
| 公開情報 | 自社サイト、プレスリリース、公開仕様書 | 個人プラン含めOK |
| 社内一般 | マニュアル、定型文、社内向け資料 | 法人プラン |
| 機密 | 見積根拠、顧客要望、契約条件 | 法人プラン＋ガイドライン |
| 極秘 | 入札戦略、知財図面、人事情報 | 原則AIに入れない、入れる場合はオンプレ／VPC |

## 4. ヒアリングで触れる質問

セキュリティ周りで聞いておくと次回提案がしやすい：

- 現在、社員が使っているAIサービスは？（個人 or 法人？）
- 情シス・法務はAI利用についてどんなスタンス？
- 過去にAI利用で社内ヒヤリ・ハット事例はあるか
- 機密度の分類ルールは存在するか
- クラウドサービス全般の利用ポリシー（外部送信制約）

> **守りすぎて何もしない／無防備に使う、どちらも経営リスク。**
>
> 正しいプランと正しいルールで、攻めながら守れる。
